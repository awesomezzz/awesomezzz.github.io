Git 是什麼 ?

簡單卻又文言的說法，Git 是一種「分散式版本的版本控制系統」。

什麼是「分散式」？什麼是「版本」？又要「控制」什麼東西？

  • 分散式

    當有人想將編輯過的舊檔案上傳到伺服器、覆蓋其他人的最新檔案時，

    系統會發出警告，因此可以避免在無意中覆蓋他人的編輯內容。

  • 版本控制

    Git 可以把檔案的狀態作為更新歷史記錄保存起來。

    因此可以把編輯過的檔案復原到以前的狀態，也可以顯示編輯過內容的差異。

能保存更新歷史記錄，可顯示編輯前後內容差異，可復原指定版本，並進行衝突管理。

工作目錄與索引

首先，先了解兩個名詞，

  • 工作目錄（Working Tree）：是保存您目前正在處理檔案的目錄，Git 相關的操作都會在這個目錄下完成。

  • 索引(Index )：位於工作目錄和數據庫之間，是為了向數據庫提交作準備的暫存區域。

  • 運作模式：（圖片引用自連猴子都能懂的Git入門指南）

    在提交至數據庫變更時，需先把變更內容加入索引中，而索引的存在也可排除工作目錄裡不必要的檔案提交，也就是並非將該工作目錄裡所有檔案全部提交。

運行基本順序

1、 git init：初始化。

2、 git status：確認狀態。

3、將不要的檔案都放入 .gitignore： .gitignore 版本控制後，讓其他夥伴也知道你把甚麼檔案放進去。

4、 git add.：將所有檔案都加入版本控制（加入資料夾）。

5、 git commit –am”版本名稱”：建立版本及版本名（命名資料夾名稱）。

6、 git branch –v：確認主幹及分支。

初階常用名詞認識

  • untracked ：不加入版本控制。

  • staged ：加入版本控制，但尚未正式建立新版。

  • conflix ：衝突。當不同分支發展不同功能後，要重新組合合併在一起時，在同個檔案有不同的地方，導致無法順利合併時，稱之。

  • hook ：發生某件事情時通知我。

  • repository ：簡稱Repo ，版本倉庫或稱版本資料庫，記錄檔案或目錄的狀態。分為遠端Repo 與本地 Repo 。

    • 遠端 Repo ：多人共享，有專用 Server。
    • 本地 Repo ：用戶個人使用，比如在自己的電腦上配置的專案倉庫（資料夾）。

  • clone ：複製遠端他人或自己的專案到本地。

  • push ：本地上傳到遠端。

  • pull ：遠端下載到本地。（ pull ＝ fetch + merge ）

  • fetch ：只取得遠端最新記錄，沒合併。

  • commit ：檔案或目錄變更、新增後，提交（Commit）以儲存到Repo 中。在Repo 產生上次與本次狀態差異記錄（Revision）。一般在 Commit 時可敘明以下 2 點，方便未來能迅速找到想復原的指定版本：

    • 修改內容的摘要
    • 修改理由

常用語法

1、git init ：初始化 git。

2、 git status：git 目前版本狀態查詢。

3、 git add：git add +檔案，將檔案加入版本控制。例：git add 123，將檔案名稱123加入版本控制。

4、 git add .：一次將目前所有檔案都加入版本控制。

5、 git rm --cached：git rm –cached +檔案，將檔案移除版本控制。

6、 git commit：新建版本（就像新建一個資料夾），輸入 git commit 後就會進入到編輯器vim，在裡面可以新增 commit message （版本敘述，就像是資料夾名稱一樣），一樣是:q!或是:wq跳回。

7、 git log：查看版本歷史紀錄。每個版本上方會寫commit 後面接一串英數交雜，這個就是版本(編)號。

8、 git checkout：git checkout + 完整的版本號。將版本更改到版本號的那個版本。

9、 git checkout master：將版本更改到最新的版本。

10、 git branch –v：確認主幹及分支。

11、 git branch + 分支名稱：新增分支並命名。

12、 git branch –d +分支名稱：刪除分支。

13、 git checkout + 分支名稱：移動到該分支。

14、 git merge +分支名稱：git merge A，將 A 分支合併進目前所在的分支。

15、 git push + 遠端主機（git hub）名稱（通常叫做 origin） + 本地分支名稱：將本地已經 commit 的版本上傳更新到 git hub。若之後有在本地新增 branch，也是一樣用此語法。

16、 git pull +遠端主機(git hub)名稱（通常叫做 origin） + 遠端分支名稱：將遠端的分支下載下來（通常發生在多人協作時，需要將別人修改過上傳到 github 的分支再次下載下來使用）。

17、 git clone +網址：將遠端的 repository 下載下來。

18、 git commit --amend：更改已經 commit 的 commit message。會進入到編輯器裡，在裡面直接更改並儲存離開。

19、 git reset HEAD^：（已經 commit）回到 commit 前的那個狀態（還保留此次修改的版本，只是尚未 commit），通常發生在 commit 後但又發現有錯需要更正內容。

20、 git reset HEAD^ --hard：（已經 commit）回到上一個版本的狀態（此次 commit 的內容完全消失）。

21、 git reset +版本號：（已經 commit ）回到該版本號的狀態（等同於放棄目前最新的狀態）。

22、 git checkout --：（尚未 commit）回到上一個版本的狀態（等同於放棄此次修改的東西）。

23、 git branch –m + 新的 branch 名稱：（必須在欲修改的那個 branch 上）修改 branch 名稱。

24、 git checkout + 遠端的 branch 名稱：將遠端的branch下載下來。
